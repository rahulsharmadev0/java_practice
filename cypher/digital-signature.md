# 📘 **Digital Signatures — Complete Guide (Concept, Need, Architecture, Algorithms, Java)**

Digital signatures are one of the most important concepts in modern cryptography.
They protect integrity, authenticity, and non-repudiation across everything from banking, payments, blockchain, email, code signing, and secure messaging.

This article breaks down digital signatures in a practical and intuitive way.

---

# 1) ⭐ **Idea (What is a Digital Signature?)**

A **digital signature** is a cryptographic mechanism where:

- The **sender** signs data using their **private key**
- The **receiver** verifies the signature using the sender’s **public key**

Formally:

```
Signature = Sign(privateKey, Hash(data))
Verify(publicKey, data, Signature) → true/false
```

### Key properties:

- **Integrity:** Data has not been changed.
- **Authenticity:** The signer is who they claim to be.
- **Non-repudiation:** The signer cannot deny signing it.

This is similar to a handwritten signature, but cryptographically secure and mathematically verifiable.

---

# 2) ⭐ **Why Do We Need Digital Signatures?**

### Without digital signatures:

- Anyone can modify data in transit.
- Anyone can impersonate anyone.
- Attackers can tamper files, code, financial instructions, or API requests.
- You cannot prove who sent what.

### Digital signatures solve these problems:

| Threat                          | Signature Protection           |
| ------------------------------- | ------------------------------ |
| Data modification               | Integrity check fails          |
| Impersonation                   | Public key mismatch            |
| Replay/fake messages            | Signer can't forge private key |
| Disputes ("I didn’t sign this") | Non-repudiation                |
| Secure API communication        | Verifiable signature           |

### Where used?

- Bank transactions
- JWT tokens
- Webhooks (Stripe, Razorpay, GitHub)
- TLS/HTTPS certificates
- Software distribution (JAR signing, npm, pip)
- Blockchain (Bitcoin, Ethereum wallets)
- Emails (S/MIME)
- Aadhaar / e-sign in India
- Licensing systems

Digital signatures are everywhere.

# 3) ⭐ **How Digital Signatures Work (Architecture)**

Digital signing is a **two-step process**:

---

## 🔶 **Step 1: Signing (Private Key)**

```
data
  ↓
hash(data)              <-- SHA-256, SHA-384, SHA-512
  ↓
sign(hash, privateKey)  <-- RSA / ECDSA / Ed25519
  ↓
signature
```

The signature contains **no public key**, only mathematically generated bytes.

---

## 🔷 **Step 2: Verification (Public Key)**

```
data + signature
   ↓
hash(data)
   ↓
verify(hash, signature, publicKey)
   ↓
true  → data authentic & unchanged
false → data tampered or wrong signer
```

The public key must be known beforehand (certificate, key file, pinned key, JWKS, etc.).

---

# 4) ⭐ **Important Clarification: Public Key is NOT inside the signature**

A signature contains **only mathematical proof**, not the public key.

The verifying party must already trust the public key from:

- Certificate Authority (CA)
- Stored public key
- JWKS endpoint
- Mobile app key pinning
- SSH known_hosts
- TLS handshake

This prevents substitution attacks.

---

# 5) ⭐ **Digital Signature Algorithms**

### 1. **RSA (older, widely used)**

- Signature = RSA private key encrypting the hash.
- Recommended: **RSASSA-PSS (SHA-256)**
  (More secure than old PKCS#1 v1.5 signatures)

### 2. **ECDSA (Elliptic Curve)**

- Smaller keys
- Faster
- Used in JWT, blockchain, TLS 1.3

Curves: `secp256r1`, `secp384r1`

### 3. **Ed25519 (modern, safest)**

- Simpler
- Fast
- Safe from common implementation bugs
- Used by SSH, Signal protocol

Java supports RSA & ECDSA natively.
Ed25519 is available via newer JDKs or BouncyCastle.

---

# 6) ⭐ **Digital Signatures vs Encryption (Difference)**

| Encryption                  | Digital Signature              |
| --------------------------- | ------------------------------ |
| Uses PUBLIC key to encrypt  | Uses PRIVATE key to sign       |
| Uses PRIVATE key to decrypt | Uses PUBLIC key to verify      |
| Goal: confidentiality       | Goal: integrity + authenticity |
| Only receiver can read      | Anyone can verify              |

They solve different problems.

# 7) ⭐ **Signature Workflow in Real Systems**

### 🔹 **JWT Tokens**

```
header.payload → sign(privateKey) → signature
```

Clients verify using the server’s public key.
Public key often delivered through `.well-known/jwks.json`.

---

### 🔹 **Bank Payment Requests (UPI, IMPS, NEFT APIs)**

Banks sign payment request body:

```
Sign(priv_bank, SHA256(requestBody))
Attach signature to header
```

Receiving bank verifies using the sending bank’s public key.

---

### 🔹 **Software Package Signing (Maven, npm, pip)**

Publisher signs JAR/Tarball.
Installer verifies before installation.

---

### 🔹 **Blockchain Transactions**

Wallet signs transaction with private key.
Nodes verify signature using wallet’s public key → ensures authenticity.

# 8) ⭐ **Java Implementation — Digital Signature (Complete Example)**

Works for any data.

---

## 🔶 **Signing**

```java
import java.security.*;
import java.util.Base64;

public class DigitalSignDemo {

    public static String sign(byte[] data, PrivateKey privateKey) throws Exception {
        Signature signature = Signature.getInstance("SHA256withRSA");
        signature.initSign(privateKey);
        signature.update(data);
        return Base64.getEncoder().encodeToString(signature.sign());
    }
}
```

---

## 🔷 **Verification**

```java
public static boolean verify(byte[] data, String signatureB64, PublicKey publicKey) throws Exception {
    Signature signature = Signature.getInstance("SHA256withRSA");
    signature.initVerify(publicKey);
    signature.update(data);
    byte[] sigBytes = Base64.getDecoder().decode(signatureB64);
    return signature.verify(sigBytes);
}
```

---

## 🔥 **Demo main()**

```java
public static void main(String[] args) throws Exception {

    // 1. Generate RSA keys
    KeyPairGenerator kpg = KeyPairGenerator.getInstance("RSA");
    kpg.initialize(2048);
    KeyPair pair = kpg.generateKeyPair();

    String message = "This is a signed message";
    byte[] data = message.getBytes(java.nio.charset.StandardCharsets.UTF_8);

    // 2. Sign
    String signature = sign(data, pair.getPrivate());
    System.out.println("Signature: " + signature);

    // 3. Verify
    boolean ok = verify(data, signature, pair.getPublic());
    System.out.println("Signature Valid: " + ok);
}
```

# 9) ⭐ **Important Best Practices**

### ✔️ Always specify charset

```
data.getBytes(StandardCharsets.UTF_8)
```

### ✔️ Use PSS for RSA signatures

```
SHA256withRSA/PSS
```

### ✔️ Do NOT embed public key inside signed message

Use trusted certificates or key distribution.

### ✔️ Use Base64 for transporting signature

Signatures contain bytes → encode safely.

### ✔️ Key rotation

Use new private keys periodically.

### ✔️ Do not ever log private keys

Only store in KeyStore, HSM, or KMS.
